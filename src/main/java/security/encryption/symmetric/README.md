# Symmetric Encryption — AES-256-CBC

One key encrypts and decrypts. Fast enough for bulk data. The key distribution problem (how to share the key securely) is solved separately by key exchange protocols (see `security.keyexchange`).

Run with:
```bash
mvn exec:java -Dexec.mainClass="security.encryption.symmetric.SymmetricEncryptionExample"
```

---

## SymmetricEncryptionExample.java

### Alice → Bob Communication

```mermaid
sequenceDiagram
    participant Alice
    participant Network as 🌐 Network (insecure)
    participant Bob

    Note over Alice,Bob: Both share the same Secret Key 🔑<br/>(must be exchanged securely in advance)

    Alice->>Alice: 1. Generate random IV (16 bytes, SecureRandom)
    Alice->>Alice: 2. AES-256-CBC encrypt(plaintext, key, IV) → ciphertext
    Alice->>Alice: 3. Base64-encode ciphertext for safe transmission
    Alice->>Network: ciphertext (Base64)  +  IV
    Network->>Bob: ciphertext (Base64)  +  IV

    Note over Network: Eavesdropper sees ciphertext + IV<br/>Useless without the shared key 🔑

    Bob->>Bob: 4. Base64-decode ciphertext → raw bytes
    Bob->>Bob: 5. AES-256-CBC decrypt(ciphertext, key, IV) → plaintext
```

### CBC Mode — How Blocks Are Chained

```mermaid
flowchart LR
    subgraph CBCChain ["🔗 CBC — Cipher Block Chaining (why IV matters)"]
        IV["Random IV<br/>(16 bytes)"] -->|"XOR ⊕"| XOR1[" "]
        P1["Plaintext<br/>Block 1"] --> XOR1 --> AES1["AES"] --> C1["Cipher<br/>Block 1"]
        C1 -->|"XOR ⊕"| XOR2[" "]
        P2["Plaintext<br/>Block 2"] --> XOR2 --> AES2["AES"] --> C2["Cipher<br/>Block 2"]
        C2 -->|"XOR ⊕"| XOR3[" "]
        P3["Plaintext<br/>Block 3"] --> XOR3 --> AES3["AES"] --> C3["Cipher<br/>Block 3"]
        NOTE["Each block depends on all previous blocks<br/>Identical plaintext blocks → different ciphertext"]
    end
```
